Every container image in this release has authenticated provenance. Stock DHI
images carry Docker's full 15-attestation suite; custom images are built with
DHI tooling and signed via Sigstore.

## Layer-by-layer attribution

The heritage stack is the place where layer attribution matters most —
each layer answers to a different patch source. Per
[ADR-0012](https://github.com/wellmaintained/packages-dhi/blob/main/docs/adr/0012-layer-python-2.7-on-debian13-base.md):

| Layer                                                    | Provenance source                                                | Signed by                          |
|----------------------------------------------------------|------------------------------------------------------------------|------------------------------------|
| debian13 rootfs + system libraries                       | DHI's SLSA provenance + DHI's `vex.dhi.json`                     | Docker, Inc. (DHI signing)         |
| CPython 2.7.18 + patch series                            | wellmaintained's SLSA provenance (BuildKit) + `python-2.7.vex.json` | wellmaintained (Sigstore via GitHub OIDC) |
| Plone 5.2.15 + SENAITE 2.6.0 (current line)              | upstream source tarballs + recorded checksums + `senaite-lims.vex.json` | (no signature on upstream sources; sources are authenticated by digest in the SBOM) |
| Deployment composition                                   | `apps/senaite-current/deployments/`                              | wellmaintained (Sigstore via GitHub OIDC) |

## Attestation sources

{{< release-images >}}

**Stock DHI images** (nginx) — attestations provided by
[Docker Hardened Images](https://www.docker.com/products/hardened-images/),
including SBOM, VEX, SLSA provenance, and 12 additional attestation types.

**Custom heritage images** (python-2.7, senaite-lims) — built using
DHI YAML definitions with attestations generated by our pipeline:
CycloneDX SBOM (scout-sbom-indexer), SPDX SBOM (syft), CVE scan (grype,
internal only — see
[ADR-0009](https://github.com/wellmaintained/packages-dhi/blob/main/docs/adr/0009-publish-vex-not-point-in-time-cves.md)),
secrets scan (gitleaks), hand-written VEX, and SLSA provenance (buildx).

## Verification

All custom heritage images are signed with [Sigstore](https://sigstore.dev)
keyless signing using GitHub Actions' OIDC identity. Verify signatures with
[cosign](https://github.com/sigstore/cosign):

```bash
cosign verify \
  --certificate-identity-regexp="https://github.com/wellmaintained/packages-dhi/" \
  --certificate-oidc-issuer="https://token.actions.githubusercontent.com" \
  ghcr.io/wellmaintained/packages-dhi/python-2.7:<release-tag>
```

```bash
cosign verify \
  --certificate-identity-regexp="https://github.com/wellmaintained/packages-dhi/" \
  --certificate-oidc-issuer="https://token.actions.githubusercontent.com" \
  ghcr.io/wellmaintained/packages-dhi/senaite-lims-current:<release-tag>
```

Stock DHI images can be verified against Docker's signing infrastructure:

```bash
cosign verify \
  --certificate-identity-regexp="https://github.com/docker/" \
  --certificate-oidc-issuer="https://token.actions.githubusercontent.com" \
  dhi.io/nginx:1.29
```
