# packages-dhi

DHI-native image pipeline for wellmaintained packages.

## Structure

- `common/tool-images.yaml` — Tool image manifest (crane, cosign, grype, syft, gitleaks, hugo)
- `common/images/` — Shared infrastructure images (minio, hugo)
- `apps/sbomify/app-images.yaml` — App image manifest (stock + custom)
- `apps/sbomify/` — sbomify application (images, deployments, release website)
- `bin/` — Tool wrapper scripts (auto-added to PATH via direnv)
- `scripts/` — Build, scan, and release scripts
- `docs/adr/` — Architecture decision records

## Prerequisites

- Docker (with buildx)
- Just
- yq

## Quick Start

```
just build minio          # Build a custom image
just build minio          # Build and produce all compliance artifacts
just extract-dhi-attestations  # Pull stock image attestations
just release-data         # Generate release website data
```

## Image Manifests

Split into spec + lock files:
- `common/tool-images.yaml` — Tool images (human-edited)
- `common/tool-images.lock.yaml` — Tool images with pinned digests (generated by `just update`)
- `apps/sbomify/app-images.yaml` — App images (human-edited)
- `apps/sbomify/app-images.lock.yaml` — App images with pinned digests (generated by `just update`)
